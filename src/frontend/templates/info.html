Agent architecture

Six distinct agents, each with its own prompt, tools, and job.

{{ icons.compass("icon") }}

Coordinator Agent

Understands the goal, decides which specialist agents to run, and merges their outputs into one confirmation.

{{ icons.shield("icon") }}

Safety & Escalation Agent

Blocks diagnosis/prescription requests and emergencies in code, and creates human-review escalations.

{{ icons.route("icon") }}

Department Routing Agent

Classifies the request and maps it to a real, active hospital department — never a hallucinated one.

{{ icons.calendar("icon") }}

Appointment Agent

Checks live slot availability and conflicts, then books, reschedules, or cancels against persisted data.

{{ icons.file_text("icon") }}

Document Agent

Classifies uploads, detects duplicates by checksum, and flags missing documents per department.

{{ icons.bell("icon") }}

Follow-up Agent

Creates appointment and document reminders, and hands off to the notification dispatcher.

AgentCare is an administrative coordination system, not a clinical one.

It never diagnoses conditions, prescribes medication, or recommends dosages. Emergencies and clinical questions are always escalated to a human. This boundary is enforced in the Safety & Escalation Agent's code, not just its prompt.

AgentCare — synthetic demo data only. Not a substitute for professional medical care.
